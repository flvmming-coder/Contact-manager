ДЕМО ВЕРСИЯ v0 — ОПИСАНИЕ БЭКЭНДА (ИСХОДНОЕ)

Архитектурная идея
В исходной DEMO v0 бэкенд реализован внутри клиентской HTML-логики (JavaScript-класс MobileContactManager). Отдельного серверного API в версии нет.

Хранение данных
- Источник хранения: localStorage браузера.
- Основные сущности:
  - contacts (массив контактов)
  - importedCount (счетчик импортированных контактов)
- При первом запуске создается стартовый набор demo-контактов.

Операции с данными
- addContact: добавление контакта.
- updateContact: обновление существующего контакта.
- deleteContact: удаление контакта с подтверждением.
- importContacts: пакетное добавление импортируемых записей.
- getFilteredContacts: фильтрация по группе + строке поиска.
- updateStats: обновление статистических показателей интерфейса.
- render/renderContactCard: формирование визуального представления данных.

Импорт
- Поддержан массовый импорт из многострочного текста (Имя,Телефон).
- Есть mock-функции импорта (например, Google/iPhone) для демонстрации потока.

Ограничения v0
- Нет SQLite/серверной базы.
- Нет аутентификации и синхронизации в реальном виде.
- Нет реального CSV/VCF парсера на уровне production.

Статус версии
DEMO v0 — это прототип бизнес-логики и UX-потока, достаточный для презентации концепции и первичной проверки требований.
