Описание версии Contact Manager v0.14.4 (16) — БЭКЭНД

Причина обновления:
Для поддержки новых функций интерфейса потребовалось расширить модель контакта, хранение данных и обработчики системных действий (фото-документ, экспорт VCF, Google-вход).

Что изменено в логике и данных:
1. Модель данных контакта:
- в структуру Contact добавлено поле `avatarPhotoUri` для хранения URI выбранной фотографии.

2. Локальное хранилище:
- ContactPrefsStorage обновлён:
  - сериализация `avatarPhotoUri` в JSON;
  - десериализация `avatarPhotoUri` из JSON;
  - совместимость со старыми записями сохранена.

3. Обработка аватаров:
- при выборе фото используется системный picker документов (`OpenDocument`);
- URI сохраняется с попыткой persistable-доступа на чтение;
- при отсутствии/ошибке URI автоматически используется буквенная заглушка.

4. Экспорт контактов:
- реализована генерация VCF payload для всех локальных контактов;
- сохранение файла через системный диалог `CreateDocument`;
- экспортируются поля: имя, фамилия, телефон, email, адрес, комментарий, день рождения (при наличии).

5. Настройки и очистка:
- добавлены обработчики действий меню настроек:
  - импорт из телефонной книги (существующий механизм перенесён в настройки);
  - экспорт VCF;
  - полная очистка контактов с подтверждением.

6. Google-режим:
- добавлен запуск сценария подключения Google-аккаунта для перехода из локального режима в сетевой (точка интеграции и сохранение выбранного аккаунта в prefs).

7. Версионирование:
- версия приложения обновлена до `0.14.4`;
- счётчик обновлений в админ-панели обновлён до `(16)`.

Проверка:
- сборка debug, lint и unit tests выполнены успешно.
