ВАРИАНТ 2. БЭКЭНД (ХРАНЕНИЕ, ЛОГИКА, ИНТЕГРАЦИИ)

Общее назначение
Бэкэнд приложения отвечает за локальное хранение данных, операции над контактами, импорт/экспорт, а в перспективе — синхронизацию и резервное копирование.

Локальное хранилище
- SQLite база данных на устройстве.
- Таблица contacts: id, имя, фамилия, телефон, email, группа, признак «рабочий», задача, адрес, день рождения, флаг импорта, даты создания/обновления.
- Все операции проходят через репозиторий, отделенный от UI.

Операции с данными
- Создание контакта.
- Обновление контакта.
- Удаление контакта.
- Получение списка контактов.
- Массовый импорт (batch insert).

Логика фильтрации и поиска
- Базовая фильтрация по группе.
- Поиск по имени, фамилии и телефону.
- Подготовка к расширению: сортировки и фильтры по частоте/дате.

Импорт и экспорт
- В DEMO реализован импорт из текстового формата.
- В планах: импорт из CSV/VCF, экспорт в CSV/VCF.

Безопасность и конфиденциальность
- Данные хранятся локально, без передачи в облако.
- Возможность добавить шифрование базы или резервных копий в следующих версиях.

Синхронизация и резервные копии (планируемо)
- Локальные бэкапы с версионностью.
- Опциональная облачная синхронизация.
- Прямой перенос между устройствами (QR-код / Wi-Fi Direct).

Нефункциональные требования бэкэнда
- Устойчивость к повреждению данных (транзакции при массовом импорте).
- Минимальная задержка при вставке/чтении.
- Совместимость с Android 5.0+.
