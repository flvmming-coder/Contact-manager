ВАРИАНТ 2. БЭКЭНД — CONTACT MANAGER v0.11

Архитектурная схема
Версия v0.11 использует локальный бэкенд на SQLite (без удаленного сервера):
- слой данных: SQLiteOpenHelper (ContactDbHelper)
- слой доступа: репозиторий (ContactRepository)
- слой UI-логики: Activity + Adapter

Структура данных
Таблица contacts:
- id (PK)
- first_name
- last_name
- phone
- email
- group_name
- is_work
- work_task
- address
- birthday
- imported
- created_at
- updated_at

Бизнес-операции
- create: добавление контакта
- read: выборка списка, выборка по id
- update: редактирование контакта
- delete: удаление контакта
- bulk import: пакетная вставка импортируемых контактов

Логика v0.11
- Если база пуста, добавляются стартовые demo-контакты.
- Подсчет статистики (всего/рабочие/импорт).
- Поиск и фильтрация на стороне клиента (по данным из локальной базы).
- Google auth реализован как UI mock (без реального OAuth).

Надежность и производительность
- Массовый импорт выполняется в транзакции.
- Время создания/обновления хранится в виде timestamp.
- Все операции выполняются локально, без сетевой зависимости.

Ограничения текущей версии
- Нет реального облачного бэкенда.
- Нет шифрования БД на уровне приложения.
- Нет полноценного импорта CSV/VCF через парсер форматов (вместо этого текстовый импорт + mock сценарии).

План следующего расширения
1. Реальный импорт CSV/VCF.
2. Экспорт контактов в файл.
3. Шифрование резервных копий.
4. Опциональная синхронизация (при подключении серверной части).
